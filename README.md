# 误触清理

[English README](./README.en.md)

面向家人手机相册的 Android 优先跨平台 MVP。应用会在本地扫描最近一批照片和视频，基于可解释的程序规则识别误触、异常与重复内容，并通过“应用内回收站”提供安全清理流程。

## 当前能力

- 申请照片与视频读取权限，并在本地完成扫描。
- 识别误触、异常、重复照片与视频。
- 展示统一识别结果列表、类型筛选、置信度、触发原因、媒体信息与预览。
- 重复结果会说明为何保留参考副本，并在卡片/预览中给出轻量对比。
- 支持自动清理：仅把高置信度候选项移入应用内回收站。
- 支持选中清理：可选择移入回收站，或在二次确认后彻底删除。
- 支持回收站恢复。
- 支持“定期清理准备”：可开启本地提醒、设置每日或每周重复提醒、在每周模式下选择星期，并展示当前排程状态与下次提醒时间。
- 支持应用内国际化：当前可在 `zh-CN / en-US` 之间切换，并持久化语言偏好；提醒文案、候选卡片、预览弹窗与主要状态文案都会跟随语言变化。
- 支持深色/浅色主题：默认跟随系统，也可在应用内手动切换 `system / light / dark`，并持久化主题偏好。

## 识别方式

首版不使用云端模型，完全在设备本地做轻量分析：

- 误触识别：综合时长、亮度、对比度、边缘信息、分辨率、文件体积。
- 异常识别：补充极短时长、异常比例、异常低分辨率、接近黑屏/无内容等规则。
- 重复识别：图片基于缩略图指纹，视频基于按时长自适应采样的多帧缩略图指纹与媒体元数据做近似分组，并保留 1 份更高质量副本。

这意味着首版更偏向“安全、可解释、先把最明显问题揪出来”，而不是追求极限召回率。

## 安全语义

- 自动清理永远不会直接永久删除媒体。
- “移入回收站”是应用内软删除：应用会隐藏这些候选项，但不会立刻从系统相册移除。
- “彻底删除”才会调用系统媒体删除能力，并且始终要求二次确认。
- “定期清理提醒”只负责提醒你重新打开应用，不会在后台偷偷扫描照片或视频，也不会在提醒触发时自动执行清理。

## 当前限制

- 首版默认扫描最近 `360` 个媒体资源，而不是整台手机的全部内容。
- 回收站是应用内状态，不是系统相册自带回收站。
- 提醒能力目前是本地通知，可配置每日或每周重复，且每周模式支持选择星期；它不是后台自动扫描或系统级定时任务。
- 识别规则仍是启发式版本，后续可以替换为更强的原生分析或模型能力；当前仓库未接入外部 AI 接口。
- 当前优先验证 Android 路径，iOS 仅保留 Expo 层兼容性。
