# iOS 风格 UI 重新设计

## 背景

根据用户反馈，当前设计的卡片式圆角风格不够美观，需要改用更适合 iOS 风格的设计。本次重新设计参考 `personal/awesome-design-md` 中的 **Apple** 和 **Linear** 设计系统。

## 设计参考

### Apple 设计风格
- **设计原则**: 消费电子产品，高端留白，SF Pro 字体，电影级图像
- **视觉风格**: 单色奢华美学，优质留白
- **适用场景**: 高端产品页面，电影级营销网站

### Linear 设计风格
- **设计理念**: 项目管理，极致简约，精准，紫色强调
- **视觉风格**: 超极简，深色界面优化
- **适用场景**: 项目管理工具，极简深色 UI，SaaS 产品页面

## 关键设计决策

### 1. 导航结构：底部 Tab 栏
采用 iOS 经典底部导航，3个 Tab：
- **📷 照片** - 主屏，显示照片网格和扫描入口
- **🗑 回收站** - 已删除项目
- **⚙️ 设置** - 应用设置

### 2. 主屏流程：相册优先
授权后显示所有照片网格（类似 iOS 照片）：
- 顶部分段控制器：[全部] [照片] [视频]
- 照片网格：3列布局，正方形缩略图
- 底部悬浮扫描按钮
- 上次清理时间记录

### 3. 扫描范围设置
在设置页使用滑块选择：
- 选项：1个月、2个月、3个月、6个月、12个月
- 默认：最近 2 个月

### 4. 扫描进度动画
- 全屏覆盖层
- 圆形进度条 + 数字计数器（如 "识别中... 50/300"）
- 当前分析文件名
- 取消按钮

### 5. 识别结果展示
- 仅显示问题照片/视频
- 长按进入选择模式
- 类型标签：误触、异常、重复
- 底部操作栏：清理选中

### 6. 详情页设计
- iOS 风格照片详情布局
- 底部标签展示异常原因
- 视频点击播放
- 重复项：左右滑动对比，选择删除

## 团队执行

依据 `docs/goal/v0.1.md` 团队定义：
- **包拯（Lead）**: 总控方向，监督进展
- **公孙策（架构）**: 架构设计，确保扩展性
- **展昭（执行）**: 底层执行，及时纠错
- **张龙、赵虎、王朝、马汉（执行小队）**: 模块执行和拆解
- **八贤王（验收）**: 高标准验收

## 非目标

1. 不接入服务端
2. 不做复杂机器学习模型
3. 首版仍保持启发式识别

## 风险与应对

1. **照片格式多样** - 分析前统一转为 JPEG
2. **性能问题** - 使用虚拟化列表（FlashList）
3. **误识别** - 展示置信度和原因，用户确认后操作
