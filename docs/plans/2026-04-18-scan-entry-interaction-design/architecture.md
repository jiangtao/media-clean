# 架构设计 - 扫描入口交互优化

## 状态机

照片页扫描入口统一为以下状态机：

1. `loading`：检查权限中
2. `denied`：未授权，展示紧凑授权入口
3. `ready`：已授权，展示扫描范围摘要与开始扫描 CTA
4. `scanning`：页内扫描轨道展示进度
5. `completed`：页内扫描轨道展示异常媒体数量与再次扫描 CTA
6. `error`：页内提示错误，但不改变整体入口区块结构

## 组件边界

### `PhotoGridScreen`

- 负责权限状态、扫描状态、结果数量和候选项管理。
- 负责把扫描范围真值、进度数据与结果数量传给扫描轨道组件。
- 负责决定当前 CTA 是授权、开始扫描还是重新扫描。

### `ScanProgress`

- 从“阻断弹窗”组件收口为“页内内嵌状态轨道”组件。
- 负责渲染：
  - 小号状态标题
  - 渐进呼吸流水线
  - 数量计数
  - 当前文件名
  - 完成结果摘要
- 不再负责 `Modal` 生命周期。

### 真值源

- `DEFAULT_SCAN_LIMIT` 作为首版扫描上限真值源，用于授权后默认范围摘要。
- 扫描进度真值仍来自 `scanMediaLibrary(..., { onProgress })`。
- 文案真值统一来自 `getAppCopy()`。

## 数据流

1. 页面聚焦时读取权限状态。
2. 已授权时显示“默认扫描最近 360 个媒体”的范围摘要。
3. 点击扫描后，页面调用 `scanMediaLibrary` 并订阅 `onProgress`。
4. `onProgress` 驱动页内扫描轨道刷新 `current/total/currentFileName`。
5. 扫描完成后，页面把 `candidateCount` 回写到入口区块，展示异常媒体数量。

## 约束

1. 页面必须继续尊重安全区，不因顶部入口改造而压到刘海或打孔危险区。
2. 顶部入口和扫描轨道必须复用同一块版心，避免完成后布局跳变。
3. 不引入新的外部状态管理库。
