# 最佳实践

[English Version](./best-practices.en.md)

## 1. 角色使用

1. Lead 不把关键阻塞步骤外包给子成员。
2. 子成员只接明确、边界清楚、可独立验证的任务。
3. 验收者不复写实现，只指出缺口并退回 Lead 裁决。

## 2. runtime 修复

1. 先补失败场景，再修实现。
2. 先查 live 路径，再查遗留路径。
3. 不以 `try/catch` 掩盖生命周期或资源释放根因。
4. 不在 runtime 未收口时继续推进体验优化。

## 3. 测试纪律

1. 先定向，后全量。
2. 新逻辑必须附测试。
3. 无法测试者必须说明原因与替代验收方式。

## 4. TODO 编排

1. 先 `P0`，后 `P1/P2/P3`。
2. 阻断项必须显式列在计划首页。
3. 每个 TODO 至少有一个验收命令。

## 5. 文档纪律

1. 标准与设计中文优先、英文互链。
2. 计划可中文单写，但必须引用对应设计与标准。
3. 文档更新要与代码现状同步。
