# Android 媒体清理 MVP 设计

## 背景

`docs/goal/v0.1.md` 要求构建一个以 Android 为优先平台的跨平台 App，用来识别、展示并清理家人手机里误触录入的照片和视频。仓库当前没有现成代码，因此本设计直接定义首个可交付 MVP 的边界、行为与实现约束。

## 设计目标

1. 扫描用户媒体库中的图片和视频。
2. 基于本地启发式规则识别“疑似误触”候选项。
3. 以安全优先的方式展示、预览、批量选择和清理候选项。
4. 默认采用“应用内回收站”假删除，避免误删。
5. 提供用户确认后的自动清理入口，并保留手动彻底删除能力。
6. 为后续定期清理提供本地提醒入口，支持每日或每周重复提醒，并在每周模式下允许选择星期，但不在后台自动扫描媒体。

## 关键决策

1. 平台与框架：采用 Expo + React Native + TypeScript。原因是本机具备 Node 环境而无 Flutter，可最快落地 Android 优先的跨平台 MVP。
2. 识别方式：首版采用本地启发式评分，不依赖云端。视频通过时长、文件体积与缩略图亮度/清晰度综合评分；照片通过亮度、对比度、边缘清晰度、分辨率与体积综合评分。
3. 清理策略：自动清理只会将高置信度候选项移入应用内回收站；手动选中清理允许用户选择“移入回收站”或“彻底删除”。
4. 安全边界：任何 destructive 行为都必须二次确认；未获取媒体权限时不展示空白页，而是展示权限说明与重试。
5. 设计方向：界面采用“家人可理解”的大卡片、低风险提示和明确 CTA，不追求复杂设计系统，优先保证可读性与安心感。

## 非目标

1. 首版不接入服务端，不做账号体系。
2. 首版不实现系统级后台扫描任务，只保留“为定期清理做准备”的最近扫描记录、自动清理入口与本地提醒。
3. 首版不做复杂机器学习模型，仅实现可验证、可解释的本地规则识别。

## 风险与应对

1. Expo 无法提供系统回收站：通过应用内隐藏列表模拟软删除；真正删除仅调用系统媒体删除接口。
2. 图片格式多样：分析前统一缩放并转为 JPEG 以降低解码复杂度。
3. 误识别风险：展示置信度、触发原因和预览入口，让用户可验证后再执行清理。
