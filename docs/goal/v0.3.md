## 目标

背景：我们家里人的手机，特别是成人的手机里面，总有一些多余的照片和视频，这些视频多来自于误触、轻易点击等
诉求：写一个App，打开后可以识别、清理误触、不小心录入的视频或相片，为手机定期清理做准备


## 进度回顾

1. 当前我们已完成了  [v0.1](./v0.1.md) [v0.2](./v0.2.md) 版本的核心功能实现

## 产品化建设

1. 我们需要进一步产品化：将流程 调整为 扫描->识别->筛选->清理->报告，更容易突出产品的逻辑等
2. 产品细节调整：
- 参考 [品牌文件夹](./cleaner-app) 下面的 promo-video.html 视频宣传步骤，对首页进行调整，设计调用impeccable 确保 clearify、明晰等
- 扫描我们进行一步的动画显示和操作，不再展示所有的相册，后台处理
- 识别：识别也在后台处理并保存在 sqlite 中（是否仍在 RN Native 需要给出详细方案），识别后有个状态显示 异常结果分类数据，然后回到我们现在的界面
- 清理：筛选和保留后，即回收站进行最终操作，回收站现在也叫 保留和清理，保留即还在原相册，清理即彻底删除
- 报告：在回收站底部记录 累计清理照片数和体积大小，根据 sqlite 进行统计
3. 引入 google firebase 监控，白屏、异常、用户操作分析等
4. 仓库文档归档和清理，所有的临时 plan 我们都先整理 在忽略，临时文档不作为交付产物.gitginore 掉

## 团都成员

我们执行成员设定，执行还是以Agent Team形式，worker 名字以拼音命名即可

1. Lead成员-包拯：负责总控项目始终监督和指导方向，完善目标初衷和进展，对其他成员进行指导反馈，要求铁面无私
2. 架构设计-公孙策：始终坚持架构的良好扩展, 将架构信息同步给各个成员，不正确反馈给 Lead，Lead 进行协调
3. 底层执行者-展昭：底层执行者要求，坚定不行的符合 成员 包拯、公孙策的安排，并及时纠错
4. 执行者小队（张龙、赵虎、王朝、马汉）：
  负责其他核心模块的执行和拆解
5. 验收者-八贤王： 负责高标准验收，有问题反馈给 Lead 成员让 Lead决策
- 对 error 零容忍，包含 runtime error 和 build error，抓取日志和及时校验
- UI轻量，确保扫描和识别过程中，不要影响 UI 线程

 及时通知其他人进行修复，之后再次验证
