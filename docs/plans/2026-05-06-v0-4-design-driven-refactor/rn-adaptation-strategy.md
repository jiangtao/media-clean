# RN 适配策略

## 背景

`light/` 与 `dark/` 目录下的设计图是 SE 尺寸稿。照片、回收站、设置三页当前实现与设计稿差异很大，本轮不能只把图当作普通视觉参考，也不能把 SE 稿像素值直接硬编码到 RN。

本文件定义 v0.4 重构时的 RN 适配策略。

## 基准

1. 视觉真值源：
   - [light](/Users/jt/places/personal/cleaner-app/preview-frames/final-screens/light)
   - [dark](/Users/jt/places/personal/cleaner-app/preview-frames/final-screens/dark)
2. 设计稿导出尺寸约为 `941 x 1672 px`，比例对应 iPhone SE 级别窄屏画布。
3. 实现基准按 SE 逻辑画布处理，不按导出像素逐项换算。

## 适配原则

1. 先还原 SE 结构：窄屏手机应优先贴近设计稿的布局、信息密度、底部操作区和视觉节奏。
2. 再做 RN 响应式映射：通过窗口尺寸、安全区和屏幕类型调整，而不是写死一套像素。
3. 媒体优先：照片网格和详情页应优先保证图片清晰、可判断、可操作。
4. 操作区稳定：底部 tab、批量操作栏、回收站确认栏必须避开手势条和安全区。
5. 设置页可扫读：Settings 在宽屏时可以增加舒展度，但不能稀释设置项分组关系。

## 实现规则

### 尺寸与间距

1. 使用 4pt spacing scale，优先 `4 / 8 / 12 / 16 / 20 / 24 / 32`。
2. 不把设计稿导出像素直接写进 `StyleSheet`。
3. 关键容器使用 `minWidth / maxWidth / flex / aspectRatio / numberOfColumns` 等 RN 约束。
4. 手机窄屏以 `width <= 390` 作为 SE 级别还原重点。
5. 宽屏手机和横屏只扩展可用内容，不整体等比放大所有文字与按钮。

### Safe Area

1. 所有主页面必须使用 `useSafeAreaInsets`。
2. 顶部标题区、底部 tab、浮动操作栏、详情页关闭按钮都必须避开危险区。
3. 回收站和筛选页的底部操作区以 `bottom inset + fixed action height` 计算内容 padding。

### Photos

1. `01-04` 是同一 Photos route 内的子态。
2. `02 scanning` 的进度布局在 SE 上贴近设计稿；其他尺寸只调整留白与容器宽度，不改变状态层级。
3. `04 filtering` 的网格列数基于可用宽度计算，SE 优先三列观感，宽屏可增加列数但保持缩略图可辨识。
4. 详情流使用可用视口尺寸计算媒体舞台，不用固定图片高度。

### RecycleBin

1. `05 recycle` 的统计条、网格、底部操作区按 SE 结构优先还原。
2. 非 SE 设备上保持底部操作稳定，网格按宽度增加列数。
3. 永久删除、恢复、保留等高风险动作必须保留清晰的空间关系。

### Settings

1. `06 settings` 的分组结构按设计稿还原。
2. 扫描范围、提醒、语言、主题、缓存清理必须保留既有业务语义。
3. 宽屏时设置页可限制内容最大宽度，不能无限拉伸成稀疏列表。

## 验收要求

1. 每个页面都要有 SE 签收：对照对应 `light/dark` SE 稿。
2. 每个页面都要有非 SE 签收：至少覆盖一类较高设备或横屏/大屏。
3. Photos、RecycleBin、Settings 必须有运行态截图证据。
4. 如果实现因 RN 运行时或安全区与设计稿存在偏差，必须在 `design-signoff.md` 记录原因。

## 不接受的实现

1. 直接把设计稿像素值写进 RN 样式。
2. 只在一种模拟器尺寸上看起来正确。
3. 用缩放整个页面的方式适配不同屏幕。
4. 为了贴图牺牲扫描恢复、通知、回收站删除等业务正确性。
