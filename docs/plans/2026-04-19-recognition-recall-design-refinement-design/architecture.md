# 架构设计

[English Version](./architecture.en.md)

## 一、当前问题分层

### 1. 识别层

1. 当前图片质量识别主要依赖 `brightness / contrast / edgeDensity` 与小文件、低分辨率规则，阈值偏极端。
2. 当前重复识别依赖单一 average hash 与近重复阈值，无法稳定覆盖：
   - 轻微压缩差异
   - 旋转 / 裁剪差异
   - 分析 fallback 场景
3. 当前没有真正的 `similar media` 层，只有 `duplicate`。
4. 当前扫描范围默认只取最近 `360` 个媒体，天然限制重复召回。

### 2. 展示层

1. 当前列表中只展示“待处理副本”，不展示代表副本，导致用户会把“识别到了 1 张”误读为“只识别出 1 张”。
2. 当前 badge / tag / detail 之间的数量与语义没有形成统一的解释模型。

### 3. 设计层

1. 扫描入口、进度反馈、完成态仍像三个独立片段。
2. 详情页底部的标签、动作和分页点层级竞争偏强。
3. 动画上仍存在状态层级同时变化带来的闪烁风险。

## 二、目标架构

### 1. Recognition Recall V2

识别链拆成四层：

1. `asset-analysis`
   - 继续复用持久化分析缓存
   - 增加多级分析结果：基础指标、blur score、主/副指纹
2. `quality-classification`
   - accidental / abnormal 增加“中度异常”层
   - 不再只抓极端坏图
3. `duplicate-detection`
   - 保留 exact / near duplicate
   - 增加 fallback retry 与双层复核
4. `similar-grouping`
   - 新增 `similar` 结果层
   - 与 `duplicate` 分开建模、分开展示

### 2. Result Semantics V2

列表与详情统一两组数量：

1. `groupTotalCount`
   - 同组媒体总数
2. `actionableCount`
   - 当前待清理或待确认数量

展示原则：

1. badge 默认显示组总数或按标准显示重复数量。
2. 详情页页码与切换项按组总数组织。
3. 操作按钮仍只作用于当前候选或用户显式选中的项。

### 3. Mobile Refinement V2

1. 扫描页采用单一卡片语义：
   - 标题
   - 范围摘要
   - 进度轨
   - 完成摘要
2. 详情页采用单舞台语义：
   - 顶部页码 / 关闭
   - 中部清晰媒体舞台
   - 底部标签行
   - 悬浮双态开关
   - 最底部分页点

## 三、关键技术决策

### 1. 质量识别从三指标升级为多指标

新增：

1. `blur score`
2. 颜色方差 / 平坦度指标
3. 更宽松的亮度与对比度区间判断

### 2. 重复识别采用双层判定

第一层：快速筛  
- 低分辨率缩图指纹

第二层：疑似复核  
- 更高尺寸缩图
- 次级指纹
- 必要的元数据兜底

### 3. fallback 不再直接放弃重复识别

当某张图分析失败时：

1. 先尝试本地重试
2. 若仍失败，保留“质量异常”判定
3. 对重复识别采用降级兜底，而不是立即退出

### 4. 扫描范围可扩展但不默认扩大

1. 默认仍维持最近 `360`
2. 预留更大范围或深扫模式
3. 范围变化必须与缓存签名关联
