## 目标

背景：我们家里人的手机，特别是成人的手机里面，总有一些多余的照片和视频，这些视频多来自于误触、轻易点击等
诉求：写一个App，打开后可以识别、清理误触、不小心录入的视频或相片，为手机定期清理做准备

## 设计和重构

### 系统设计

1. 当前我们在 [./docs/v0.1.md] 已经具备完备的能力，但存储设计等不符合

#### 存储设计

1. 我们对扫描后的文件进行 hash 存储，id，使用长期存储
2. 扫描后异常的 存储 status 0 是正常(未扫描) 1 保留下来 -2是删除 -1 是 error 需要备注 error 扫描后的 error信息， link_id 相关的重复 id
3. 我们的列表和回收站都从 步骤 2 的存储里面识别出来

### 扫描识别

1. 我们的识别现在不够灵活，需要有更好的设计灵活，给出方案
2. 扫描可以托盘扫描，最小化之后也继续扫描，扫面后更新视图即可

### 通知

1. 最小化扫描后通知用户扫描完成，打开可以查看进展
2. 其他通知：在设置定时后，后台运行检查，如果有新增的媒体提示用户重新扫描

## 团都成员

我们执行成员设定，执行还是以Agent Team形式，worker 名字以拼音命名即可

1. Lead成员-包拯：负责总控项目始终监督和指导方向，完善目标初衷和进展，对其他成员进行指导反馈，要求铁面无私
2. 架构设计-公孙策：始终坚持架构的良好扩展, 将架构信息同步给各个成员，不正确反馈给 Lead，Lead 进行协调
3. 底层执行者-展昭：底层执行者要求，坚定不行的符合 成员 包拯、公孙策的安排，并及时纠错
4. 执行者小队（张龙、赵虎、王朝、马汉）：
  负责其他核心模块的执行和拆解
5. 验收者-八贤王： 负责高标准验收，有问题反馈给 Lead 成员让 Lead决策
- 对 runtimeError 零容忍，抓取日志和及时校验
- 对 build error 零容忍

 及时通知其他人进行修复，之后再次验证
