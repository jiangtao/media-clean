# 最佳实践

[English Version](./best-practices.en.md)

## 一、识别侧

1. 先补失败测试，再调阈值；不要凭感觉改分数。
2. 质量识别必须区分：
   - 极端异常
   - 中度低质量
   - 正常媒体
3. 重复识别必须区分：
   - 完全重复
   - 近重复
   - 相似
4. 单次分析 fallback 不能直接等价于“无法比较”。
5. 任何新增指纹或复核步骤，都必须复用已有持久化缓存签名，防止扫描耗时失控。

## 二、展示侧

1. 列表数量与详情页码必须说同一件事。
2. “保留代表副本”与“展示组总数”必须分开建模，不能混在一个数字里。
3. badge 说数量，tag 说原因，回收站说去向，三者不要混义。

## 三、设计侧

1. 品牌词优先级：简洁 > 利落 > 可信 > 本地化 > 流畅。
2. 流畅不是炫技，而是连续、稳定、不闪。
3. 扫描页与详情页都应优先让媒体和结果成为主角，不让辅助说明抢层级。
4. 底部辅助信息必须收紧，不得靠堆面积制造“设计感”。

## 四、质量门槛

1. 每一个识别规则改动都必须补失败样本。
2. 每一个详情或扫描交互改动都必须补回归测试。
3. 任何波次结束前必须过：
   - `npm run typecheck -- --pretty false`
   - `npm run test -- --run`
